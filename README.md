# 🤖 Бот управления очередями на сдачу лаб

Telegram-бот для управления очередями студентов на сдачу лабораторных работ по предметам **ОАИП**, **ЧМ**, **АИСД**. Поддерживает две подгруппы.

---

## 📁 Структура проекта

```
tg_queue_bot/
├── bot.py           # Основной код бота
├── students.json    # База студентов (ID → имя, фамилия, подгруппа)
├── queues.json      # Очереди (создаётся автоматически)
├── requirements.txt
├── Procfile         # Для Railway
├── railway.toml     # Конфиг Railway
├── .env.example     # Пример переменных окружения
└── .gitignore
```

---

## ⚙️ Шаг 1 — Создание бота в Telegram

1. Открой [@BotFather](https://t.me/BotFather) в Telegram
2. Напиши `/newbot`
3. Придумай имя и username боту (username должен заканчиваться на `bot`)
4. Скопируй **токен** — он понадобится для `BOT_TOKEN`
5. Узнай свой Telegram ID через [@userinfobot](https://t.me/userinfobot) — это `ADMIN_ID`

---

## 👥 Шаг 2 — Заполнение базы студентов

Отредактируй `students.json`. Ключ — это **Telegram User ID** студента:

```json
{
  "123456789": {
    "name": "Иван",
    "surname": "Иванов",
    "subgroup": 1
  },
  "987654321": {
    "name": "Мария",
    "surname": "Петрова",
    "subgroup": 2
  }
}
```

> Как узнать ID студента? Добавь [@userinfobot](https://t.me/userinfobot) в группу,
> или попроси студентов написать ему лично — он ответит их ID.

---

## 🚀 Шаг 3 — Деплой на Railway

### 3.1 Загрузи проект на GitHub

```bash
git init
git add .
git commit -m "init"
git remote add origin https://github.com/ВАШ_ЛОГИН/ВАШ_РЕПО.git
git push -u origin main
```

> ⚠️ Убедись, что `.env` и `queues.json` не попали в коммит (они в `.gitignore`)

### 3.2 Создай проект на Railway

1. Зайди на [railway.app](https://railway.app) → **New Project**
2. Выбери **Deploy from GitHub repo**
3. Выбери свой репозиторий
4. Railway автоматически обнаружит `Procfile` и установит зависимости

### 3.3 Добавь переменные окружения

В Railway → вкладка **Variables** → добавь:

| Переменная  | Значение                  |
|-------------|---------------------------|
| `BOT_TOKEN` | токен от BotFather        |
| `ADMIN_ID`  | твой Telegram User ID     |

### 3.4 Добавь бота в группу

1. Открой группу → **Добавить участника** → найди своего бота
2. **Важно:** в настройках группы → **Администраторы** → НЕ делай бота админом
   (он работает как обычный участник)
3. Если группа публичная — всё готово
4. Если группа приватная — убедись, что в настройках группы разрешено читать сообщения участникам

## 🗣️ Как пользоваться

### Для студентов

Написать в группу любую фразу с триггером:

```
занимаю место на оаип
займу место на чм
записываюсь на аисд
```

Бот автоматически:
- Определит предмет по ключевому слову
- Определит подгруппу по Telegram ID
- Добавит в очередь FIFO с временем записи

**Принудительная запись (если бот не распознал сообщение):**
```
/add_oaip   — записаться на ОАИП
/add_chm    — записаться на ЧМ
/add_aisd   — записаться на АИСД
```

**Просмотр очередей:**
```
/queue
```

---

### Для администратора

**Удалить конкретного студента из очереди:**
```
/remove оаип 1 123456789
/remove чм 2 987654321
```

**Принудительно записать другого студента:**
```
/add_oaip 123456789
/add_chm 987654321
```

**Очистить все очереди сразу:**
```
/clear_all
```

**Очистить все очереди одной подгруппы:**
```
/clear_sub 1
/clear_sub 2
```

**Очистить очереди конкретного предмета:**
```
/clear_subject оаип          — обе подгруппы
/clear_subject чм 1          — только подгруппа 1
/clear_subject аисд 2        — только подгруппа 2
```

## ❓ Частые вопросы

**Бот не реагирует на сообщения в группе?**
- Убедись, что бот добавлен в группу
- В приватных группах: Settings → **Members can send messages** должно быть включено
- Проверь переменную `BOT_TOKEN` в Railway

**Студент не может записаться?**
- Его Telegram ID должен быть в `students.json`
- После добавления ID нужно сделать новый коммит и Railway автоматически передеплоит

**Очереди сбрасываются после передеплоя?**
- Это нормально для Railway без Volume. Для постоянного хранения:
  добавь Railway Volume (Volumes → New Volume → примонтируй к `/app`)
  и измени путь: `QUEUES_FILE = "/data/queues.json"`
